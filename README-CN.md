## 💡 产品哲学：克制，至纯，唯控制论

### 1. 克制，是最核心的设计原则

每添加一个功能，我们都会严苛地审视：*“这个东西真的是必要的吗？”*
Open Agent 从不替你做决定，不预设你的流派，更不会在暗中注入任何冗余的逻辑。它只负责提供坚固、纯粹的基础设施。剩下的，全部交由你来定义。

### 2. 拒绝预设人设——除非你自己写

Open Agent **不内置任何系统提示词（System Prompt）**。
当你开启一个新会话时，AI 没有记忆包袱，没有刻板风格，它是一张白纸。你只需在工作目录下创建一个 AGENTS.md，写下你的律令，AI 才会拥有“灵魂”。
*没有这个文件，它只是一个冷静的、等待被你塑造的工具。*

### 3. 100% 透明度：拒绝黑箱

在 Open Agent 中，AI 的每一个思考步骤都毫无保留地暴露在你的面前：

* 它是如何拆解问题的？
* 它调用了什么工具？
* 传入了什么参数？
* 拿到了什么返回结果？

这种极致的透明不是一种功能，而是一种态度：**作为造物主，你应当洞悉你助手的一切行为。**

### 4. 代码极简，架构裸奔

整个项目的代码量克制到了极致，目录结构一目了然。
没有过度设计的抽象层，没有套娃式的架构，每一个文件都职责单一。**少，意味着极低的理解成本、极高的可定制性，以及无法动摇的信任感。**

### 5. 独裁般的最终控制权

在 Open Agent 的世界里，你拥有绝对的统治力：

* **系统提示词**完全由你手写
* **模型提供商**与具体模型由你自由盲选和切换
* **工作目录**被死死限制在你指定的疆域内
* **定时任务**的时机和内容由你全权指派
* **任何时刻**，你都可以一键强行中断 AI 的执行

> ⚠️ **Open Agent 是你的工具，不是你的代理。它为你工作，绝不替你工作。**

## ⚡ 核心能力

* 💬 **纯粹对话 (Chat)**
  与 AI 进行毫无杂质的自然语言交互。每一次对话都有完整的历史上下文快照，支持随时回溯、分支继续或物理粉碎。
* 📂 **精准文件操作 (File IO)**
  AI 可严格按照你的指令，在**指定工作目录下**进行高精度的文件创建与流式编辑。绝不越界，安全可控。
* 💻 **原生命令执行 (Shell Execution)**
  赋予 AI 操作系统级的能力。Windows 下原生调用 PowerShell，Linux/macOS 下调用 Bash，完美闭环自动化工作流。
* 🧩 **模块化技能 (Skills)**
  通过 SKILL.md 文件抽象并复用你的专属能力模块。让 AI 在特定场景下精准组装并调用不同技能。技能由你亲手编写，由你彻底掌控。
* 🌐 **MCP 协议全生态支持 (Model Context Protocol)**
  无缝连接外部工具与生态服务。完美支持 HTTP、SSE (Server-Sent Events) 以及 stdio 三种主流连接管道。
* ⏱️ **精确时钟 (Cron Tasks)**
  支持标准 Cron 表达式。无论是每日复盘报告、定时系统巡检，还是周期性资产流转，让 AI 严格按你的生物钟自动运转。
* 🤖 **多模型自由穿梭 (Multi-Model)**
  同时配置多个模型供应商（OpenAI, Anthropic, Gemini, DeepSeek 等），在不同任务场景下，一键切换最合适的“大脑”。

## 🛠️ 适用场景

* **控制狂的福音：** 你无法忍受商用 AI 软件各种不知底细的内置 Prompt 和暗度陈仓的后台操作。
* **重度自动化玩家：** 你需要一个能完美理解你的意图，且能老老实实帮你操作本地文件、跑脚本的 Agent。
* **冷酷的极简主义者：** 你厌恶了花里胡哨的 UI 和复杂的配置，只想回归最纯粹的“输入-输出-执行”闭环。
* **全栈/AI 开发者：** 你想基于一个极其轻量、清晰的开源底座，快速魔改出属于自己的 AI 终极终端。
